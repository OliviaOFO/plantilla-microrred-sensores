## 💻 Flujo de Trabajo Sugerido para el Grupo

A diferencia de un repositorio tradicional, GitHub Classroom ya les provee un entorno privado exclusivo para su grupo. Se recomienda el siguiente flujo de trabajo en su computadora local:

1. **Clonar el repositorio:**
   ```bash
   git clone <enlace-de-este-repositorio>
   cd <nombre-de-la-carpeta>
   ```

2. **Iniciar el desarrollo:**
   Renombren el archivo `PLANTILLA_ENTREGA.md` a `MEMORIA_TECNICA.md` (o trabajen directamente sobre la plantilla) y comiencen a completar los 13 puntos exigidos por la cátedra utilizando formato Markdown.

3. **Subir avances periódicamente:**
   Eviten subir todo el último día. Mantengan un registro claro de quién trabaja en cada sección haciendo commits limpios:
   ```bash
   git status
   git add .
   git commit -m "Añadidos planos de planta y justificación del protocolo de comunicación"
   git push origin main
   ```

4. **Entrega Final:**
   Al llegar a la fecha límite, el estado del archivo principal en la rama `main` será considerado como la entrega definitiva del grupo. El docente revisará el historial de commits para evaluar la equidad del trabajo en equipo.
